Perubahan Struktur Pricing:

  • Sebelumnya: Lapangan memiliki satu field harga_per_jam
  • Sekarang: Lapangan memiliki dua field harga_pagi (06:00-11:59) dan harga_malam (12:00-23:59)
  • Benefit: Sistem dapat menerapkan pricing dinamis berdasarkan waktu booking
  • Perhitungan: Jika booking melewati boundary (12:00), harga akan dihitung proportionally:
    • Contoh: Booking 11:00-13:00 dengan harga_pagi=80.000 dan harga_malam=100.000
    • Harga = (1 jam æ 80.000) + (1 jam æ 100.000) = 180.000

HTTP Status Codes

Status Code Kategori Arti Contoh Kasus
200 OK Success Request berhasil, response normal GET, PATCH, DELETE berhasil
201 Created Success Resource baru berhasil dibuat POST register, create booking, create lapangan
400 Bad Request Client Error Request tidak valid / parameter salah format Validation error, missing required field, invalid format
401 Unauthorized Client Error Token tidak valid atau expired / tidak authenticated Token missing, token invalid, token expired
403 Forbidden Client Error Tidak punya akses (bukan admin, dll) User biasa coba akses endpoint admin
404 Not Found Client Error Resource tidak ditemukan Lapangan ID tidak ada, booking ID tidak ada, user tidak ada
409 Conflict Client Error Conflict dengan data yang sudah ada Email sudah terdaftar, lapangan sudah dibooking di jam tersebut
500 Server Error Server Error Error di server (unexpected) Database error, server crash, unexpected exception

Payment Status Values

Status Deskripsi Action Tersedia Durasi/Deadline
"pending" Booking baru, menunggu konfirmasi pembayaran Confirm Payment, Cancel 15 menit untuk confirm
"confirmed" Pembayaran sudah dikonfirmasi, booking valid Cancel Valid sampai booking time selesai
"expired" Booking sudah expired (15 min passed tanpa confirm) - Status final (tidak bisa di-undo)
"cancelled" Booking dibatalkan oleh user atau admin - Status final (tidak bisa di-undo)

ÞÀ Tips, Tricks & Best Practices

Þ Security Tips

  • Jangan hardcode API URL di kode - gunakan environment variables (.env)
  • Simpan token JWT di localStorage atau secure cookie
  • Validasi input di frontend sebelum kirim ke API
  • Handle token expiry dengan cara re-login otomatis atau redirect ke login page
  • Jangan log atau display token atau password di console
  • Gunakan HTTPS di production (bukan HTTP)

Þ₤ Testing Tips

  • Test dengan Postman atau VS Code REST Client extension
  • Gunakan environment variables untuk token & ID
  • Test error cases juga (validation error, unauthorized, dll)
  • Catat token & ID untuk reuse di request berikutnya
  • Cek Network tab di browser DevTools untuk debugging

ÞÝ Frontend Implementation Tips

  • Gunakan axios interceptor untuk auto-add Authorization header
  • Handle loading state saat API request ongoing
  • Display countdown timer saat booking pending payment
  • Show user-friendly error messages dari API response
  • Implement auto-retry untuk network error dengan exponential backoff
